# TOEIC 単語ドリル 設計指示書（AI向け）

このフォルダは、AIが本プロジェクトの仕様・設計を理解し、修正・機能追加を行う際の参照用です。

## ドキュメント一覧

| ファイル | 内容 |
|----------|------|
| [01-OVERVIEW.md](01-OVERVIEW.md) | プロジェクト概要・目的・技術スタック・制約 |
| [02-ARCHITECTURE.md](02-ARCHITECTURE.md) | アーキテクチャ・画面構成・主要ファイルの責務 |
| [03-DATA.md](03-DATA.md) | データモデル・永続化・CSV形式・音声ファイル規則 |
| [04-FEATURES.md](04-FEATURES.md) | 機能一覧と実装の要点（抽出ルール・マスキング・音声・インポート/エクスポート等） |

## 利用のしかた

- **既存挙動の理解**: 02 → 03 → 04 の順で読むと、画面・データ・機能の流れが把握しやすい。
- **修正・追加時**: 該当する機能が 04-FEATURES のどの項目か確認し、03-DATA のスキーマ・CSV形式と矛盾しないようにする。
- **ルートの要件書**: プロジェクトルートの `docs/markdown.md` に、要件リストとタスク一覧がある。設計書と突き合わせて実装する。

## 用語

- **フレーズ**: 1件の英単語＋日本語訳＋英語例文のまとまり（Phrase モデル1件）。
- **正解/不正解**: 画面上の正解・不正解ボタンで、連続正解回数（correctCount）を更新する操作。
- **日単位**: 午前5時区切りで1日。ステータスバーの出題数・正答率はこの「日」で集計。
